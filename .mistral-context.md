# Project Context: Laravel 12 High-Performance Agentic Stack

## Core Environment
- **PHP:** 8.4.17 (Utiliser les Property Hooks et les types de retour isolés)
- **Framework:** Laravel v12 (Structure de fichiers moderne)
- **Runtime:** Laravel Sail (Docker-based)
- **OS Target:** Linux (Alpine based)

## Frontend & Styling
- **Stack:** TALL Lite (Tailwind CSS v3 + Alpine.js v3)
- **Starter Kit:** Laravel Breeze v2 (Inertia ou Blade selon le contexte du controlleur)
- **CSS:** Tailwind CSS v3 (JIT mode actif)

## Tooling & Quality
- **Linter:** Laravel Pint v1 (Style de code Laravel par défaut)
- **Testing:** PHPUnit v11 (Utiliser les attributs PHP 8 pour les tests, pas les annotations @test)
- **CLI Interaction:** Laravel Prompts v0 (Utiliser pour toute commande artisan personnalisée)
- **Integrations:** Laravel MCP v0 (Model Context Protocol - prêt pour les interactions agentiques)

## Strict Coding Rules
1. **Pint Compliance:** Toujours générer du code qui passe `vendor/bin/pint`.
2. **PHP 8.4 Features:** Utiliser les `Property Hooks` pour les getters/setters simples.
3. **Type Safety:** Déclaration stricte des types (`declare(strict_types=1);`) et typage complet des propriétés.
4. **Laravel 12 Syntax:** Utiliser les nouvelles façades et les méthodes Helper simplifiées de la v12.
5. **Testing:** Toujours inclure un test PHPUnit v11 pour chaque nouvelle fonctionnalité.

## Instructions pour l'Agent
- Lors de la création de commandes Artisan, utilise impérativement `laravel/prompts` pour l'interactivité.
- Pour les intégrations IA, utilise les spécifications `laravel/mcp`.
- Garde les composants Alpine.js légers et inline autant que possible pour Breeze.
